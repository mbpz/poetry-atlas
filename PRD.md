# 中国古诗词地图 — 产品需求文档（PRD）

> **在地图上阅读中国，在诗词中穿越历史。**

---

# 一、产品定位

## 1.1 一句话定位

> **以"空间+时间"作为入口，而不是以作者、朝代、诗名作为入口。**

用户浏览的是中国，而不是诗集。

## 1.2 产品形态

用户打开网站之后：

- **不是看到列表**
- **而是看到一张中国地图**

例如：

```
———————————————
          中国
      （全屏地图）

      ● 长安（312首）

● 成都（281首）

        ● 庐山（186首）

  ● 黄鹤楼（128首）

      ● 西湖（420首）

———————————————
```

鼠标悬停：

```
成都

唐代：126首
宋代：89首
明代：32首
......

点击进入
```

整个浏览方式类似 **Google Earth**。

# 二、目标用户

| 用户类型 | 场景 | 核心诉求 |
| -------- | ---- | -------- |
| 普通诗词爱好者 | 休闲浏览 | 发现与地点相关的诗词，感受文化氛围 |
| 中小学生 | 语文学习 | 通过地图理解诗词背景，辅助课堂学习 |
| 教师 | 教学展示 | 直观讲解诗人行迹、文学地理 |
| 旅行者/文旅爱好者 | 出行规划 | 按诗词规划文化旅行路线 |
| 数字人文研究者 | 研究分析 | 探索文学中心迁移、诗人交游网络 |

# 三、核心需求分析

整个产品建议分四层。

---

## 第一层：地图（Where）—— 产品入口

全国地图。

支持层级下钻：

> 全国 → 省 → 市 → 县 → 具体地点

例如：

```
中国
  └── 湖北
       └── 武汉
            ├── 黄鹤楼
            ├── 晴川阁
            ├── 蛇山
            └── 东湖
```

最后定位到一个具体地点：

```
岳阳楼
  共有诗词：876首
  作者：142人
```

### 功能要点

- 地图支持缩放、拖拽、下钻
- 每个地点以气泡/标记显示诗词数量
- 悬停显示朝代分布
- 点击进入地点详情页

---

## 第二层：时间（When）—— 朝代筛选

时间轴：

```
先秦 → 汉 → 魏晋 → 南北朝 → 隋 → 唐 → 五代 → 宋 → 元 → 明 → 清 → 近现代
```

交互行为：

- 点击 `唐`，地图整体刷新为唐代诗词分布
- 原本杭州显示"520首"，切到唐代后显示"128首"
- 再切到宋代，立即变为"391首"
- 整个地图形成一种**文化热力图**

### 功能要点

- 顶部朝代时间轴，支持单选/多选
- 切换后地图动画过渡
- 显示当前朝代的全国诗词总量与分布概览

---

## 第三层：人物（Who）—— 作者筛选

时间确定后，可继续筛选作者：

```
全部 / 李白 / 杜甫 / 苏轼 / 陆游 / 白居易 / 辛弃疾
```

示例：

- 只看 **李白** → 地图变成「李白旅行地图」
- 用户可直观看到：四川 → 湖北 → 安徽 → 江苏 → 山东

### 功能要点

- 支持按作者筛选地图数据
- 显示作者活动轨迹与轨迹线
- 对比模式：支持多作者同时展示

---

## 第四层：诗词（What）—— 内容详情

点击地点后，例如 `黄鹤楼`，右侧展示全部相关诗词：

```
┌────────────────────────────────────────┐
│ 《黄鹤楼送孟浩然之广陵》 — 李白         │
│  故人西辞黄鹤楼，烟花三月下扬州。        │
│  孤帆远影碧空尽，唯见长江天际流。        │
├────────────────────────────────────────┤
│ 《黄鹤楼》 — 崔颢                       │
│  昔人已乘黄鹤去，此地空余黄鹤楼。        │
│  黄鹤一去不复返，白云千载空悠悠。        │
├────────────────────────────────────────┤
│ 《登黄鹤楼》 — ……                       │
│  ……                                    │
└────────────────────────────────────────┘
```

每首诗词支持：

- 全文
- 注释
- 译文
- 朗读（AI 语音）
- 创作背景
- 作者信息
- 地图位置
- 历史图片

# 四、地图层级设计

建议不局限于行政区，而是建立五大地层。

---

## ① 行政地名

```
北京 / 南京 / 杭州 / 苏州 / 成都 / 西安
```

## ② 山川

```
泰山 / 华山 / 黄山 / 庐山 / 武夷山 / 衡山
```

## ③ 江河湖泊

```
长江 / 黄河 / 湘江 / 钱塘江 / 洞庭湖 / 太湖
```

## ④ 名胜古迹

```
黄鹤楼 / 岳阳楼 / 滕王阁 / 寒山寺 / 西湖 / 瘦西湖 / 灵隐寺 / 赤壁
```

## ⑤ 古地名

**这是最重要的一层。**

```
长安 / 临安 / 姑苏 / 建康 / 洛阳 / 金陵 / 汴京 / 幽州 / 益州
```

现代地图需要支持**古 ↔ 今自动映射**：

```
长安    ↓    西安
姑苏    ↓    苏州
临安    ↓    杭州
```

> **提示：这是数据库建设中工作量最大的部分。**

# 五、地图展示模式

建议支持四种可视化模式。

---

## 模式 1：气泡图（Bubble Map）

```
      ○ 北京

   ◎ 西安

        ◎◎ 杭州

 ○ 成都
```

- 气泡大小代表诗词数量
- 适合快速感知文学密度分布

## 模式 2：热力图（Heatmap）

- 红色：诗词高密度区（杭州、西安、洛阳、苏州）
- 蓝色：诗词低密度区（新疆、西藏）
- 直观呈现"文学中心"

## 模式 3：诗词云（Poetry Cloud）

- 缩放到具体地点（如西湖）
- 地图上浮现金句：
  ```
  欲把西湖比西子
  山外青山楼外楼
  接天莲叶无穷碧
  ```
- 呈现艺术化、氛围感

## 模式 4：作者轨迹（Author Trajectory）

- 选择作者（如李白）
- 地图绘制旅行路线：
  ```
  四川 → 重庆 → 湖北 → 安徽 → 南京 → 山东
  ```
- 点击路线节点展示途中所作诗词
- 类似旅游路线图

# 八、数据库设计

核心模型：

```
Dynasty（朝代）
├── id
├── name
├── start_year
├── end_year

Author（作者）
├── id
├── name
├── dynasty_id
├── birth_place

Poem（诗词）
├── id
├── title
├── content
├── author_id
├── year

Place（地点）
├── id
├── name
├── type（城市/山川/古地名/景点）
├── longitude
├── latitude
├── modern_name
├── parent_place

PoemPlace（诗词-地点关联）
├── poem_id
├── place_id
├── relation_type（创作地/描写地/经过地/送别地等）
├── confidence（关联可信度）
```

### 设计要点

- **Poem ↔ Place 为多对多关系**：一首诗可能涉及多个地点（如送别诗涉及出发地、目的地）
- **relation_type**：区分创作地、描写地、经过地、送别地等
- **confidence**：关联可信度，处理古诗中存在的历史考据差异
- **Place.modern_name**：古地名到现代地名的映射字段
- **Place.parent_place**：支持地点层级嵌套

# 十、产品愿景

建议将产品定位为一个**数字人文平台（Digital Humanities）**，而不是单纯的古诗词检索网站。

它应融合四个维度：

- **空间维度**：地图、地形、古今地名、旅行路线
- **时间维度**：朝代演变、历史事件、文化中心迁移
- **人物维度**：诗人关系网络、人生轨迹、交游圈
- **文学维度**：诗词、名句、意象、典故与注释

最终形成一个**可探索、可学习、可研究**的「中国诗词时空知识图谱」。

无论是普通用户、学生、教师，还是文旅机构和数字人文研究者，都可以从不同入口获得价值。
